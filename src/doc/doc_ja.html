1. SgSwingとは

'SgSwing'は，プログラミング言語Sangoで作成されたプログラムからJava Swingを使用するインターフェースルーチンです。 AKIYAMA Isaoが趣味で開発しています。

SgSwingは，Swingの機能のうち，よく使用される機能を提供します。 API(Application Programming Interface)は，Swingのそれとほぼ対応していますが，そうでないものも一部あります。

2. インストール

Sango1.1以上，Java1.8以上がインストールされている必要があります。

［目次］

2.1. Linux環境へのインストール

以下の手順でインストールします。

  1. リリースファイルを解凍します。
  2. 解凍してできた「release」をシェルのカレントディレクトリにします。
  3. 「unix-configure.sh」にインストール先ディレクトリを絶対パスで指定して実行します。 ここでは「/opt/sgswing」をインストールディレクトリとします。 正常終了後，「install.sh」を実行します。
    4. 実行例（初期インストール） $ tar -xz -f sgswing1.0.0_1.tgz $ cd release $ ./unix-configure -install-to /opt/sgswing $ sudo ./install.sh
    実行例（更新インストール） $ tar -xz -f sgswing1.0.0_1.tgz $ cd release $ ./unix-configure -install-to /opt/sgswing $ sudo ./install.sh -mode upgrade
  4. インストールしたライブラリファイルをsangoc，sangoのモジュール探索パスに追加し，動作確認をします。 モジュール探索パスへの追加は以下のいずれかの方法で行います。 ここでは「/opt/sango」をSangoのインストールディレクトリとして説明します。
    • （方法1）Sangoライブラリディレクトリ配下にSgSwingライブラリディレクトリへのリンクを張る
      実行例（設定と動作確認） $ sudo ./link-lib.sh /opt/sango $ cd /opt/sgswing/sample $ sango hello
    • （方法2）Sangoライブラリディレクトリ配下にSgSwingライブラリをコピーする
      実行例（設定と動作確認） $ sudo ./copy-lib.sh /opt/sango $ cd /opt/sgswing/sample $ sango hello
    • （方法3）sangoc，sangoの環境変数SANGO_MODULESにSgSwingライブラリディレクトリを指定する
      実行例（設定と動作確認） $ export SANGO_MODULES=/opt/sgswing/lib:$SANGO_MODULES $ cd /opt/sgswing/sample $ sango hello
    • （方法4）sangoc，sangoの-modules（-m）オプションにSgSwingライブラリディレクトリを指定する
      実行例（動作確認） $ cp /opt/sgswing/sample/hello.sg . $ sangoc -m .:/opt/sgswing/lib hello.sg $ sango -m .:/opt/sgswing/lib hello
  5. 解凍してできたディレクトリは不要ですので，削除します。

3.3. コンテキスト

SgSwingはSwingとの橋渡しのため，「コンテキスト」（<context_h>）を使用します*1。 コンテキストを誤って使用すると，プログラムのフリーズなどを引き起こす可能性がありますので，本節，およびリスナの使用方法を理解した上でプログラムを作成してください。

コンテキストには，「イニシエータ・コンテキスト」(initiator context)と「コールバック・コンテキスト」(callback context)があります。

イニシエータ・コンテキストは，プログラム開始時に，GUI部品を設定するために使用します。 イニシエータ・コンテキストは，initiator_context（別名icx）で取得します。 何回取得しても同一のオブジェクトが得られます。

GUIを表示しSwingが稼働するとイニシエータ・コンテキストは不要になりますが，コンテキスト・オブジェクトは有効であり，使用してSgSwingを呼び出すこともできます。 この場合は，コールバックと非同期にSwingに要求を行います。

コールバック・コンテキストは，リスナの実行に対応しています。 Swingからコールバックが発生するとSgSwingはコールバック・コンテキストを生成し，対応するユーザ定義のリスナ・ルーチンに渡します。 リスナがSgSwingを呼び出す場合は，開始時に渡されたコンテキストを使用しなければなりません。 リスナが終了すると，当該コンテキストは破棄され，コンテキスト・オブジェクトは無効になります。

*1：SgSwingは，コンテキストにしたがい，Swing機能の呼び出しを，Swingのイベント・ディスパッチ・スレッド（event dispatch thread）で実行させます。

［目次］

3.4. データ型

この節では扱うデータの型と相互変換について，例を交えて解説します。

SgSwingは，Swingのオブジェクト*1（以降，Swingオブジェクトと呼びます）への参照をハンドルとして提供します。 ハンドルの型は，Swingオブジェクトの型に応じた型になっています。 例えばjavax.swing.JButtonへのハンドルは<button_h>です。 JButtonはjavax.swing.AbstractButtonの派生クラスですので，JavaプログラミングにおいてはAbstractButtonとしても使用できます。 SgSwingはAbstractButtonに対して<a_abutton_h>をハンドルとしますが*2，Sangoには型の継承という概念がないため，<button_h>は<a_abutton_h>としては扱えません。 この場合は，button_as_abutton（別名as_abutton）を使用して，<button_h>を<a_abutton_h>に変換します。

button_as_abuttonの定義 eval <button_h> *Button button_as_abutton | as_abutton @public -> <a_abutton_h> ;;

これと同様に，基底クラスにキャストする関数が適宜提供されています。 これらの関数はSwingの機能を使用しないため，コンテキストは不要です。

次に，基底クラスを派生クラスにキャストする例を説明します。

フレームからコンテントペインを取得するとパネル（<a_panel_h>）が返されます。 SgSwingでは，パネルを抽象クラス（インスタンス生成できないクラス）として位置づけています*3 *4。

frame_get_content_paneの定義 eval <context_h> *Cx <frame_h> *Frame frame_get_content_pane | get_content_pane @public -> <a_panel_h> ;;

フレームのコンテントペインは，初期状態では，BorderLayoutを持つパネル（<border_laytout_panel_h>）です。 そこで<a_panel_h>を<border_laytout_panel_h>にキャストする必要があります。 そのためには，次の関数を使用します。

a_panel_instanceの定義 eval <a_panel_h> *Panel a_panel_instance | instance @public -> <swing_obj> ;;

<swing_obj>は，具体的な（最終的な具象）クラスのインスタンスに対応するハンドルを持ちます。 返された値をデータ構築子で選択すれば，ハンドルを取得できます。

swing_objの定義 data <swing_obj> @public := | <frame_h> frame_obj$ ... | <border_layout_panel_h> border_layout_panel_obj$ | <grid_layout_panel_h> grid_layout_panel_obj$ ... ;;

オブジェクトの実際の型がわかっていれば，<swing_obj>を特定のハンドルに落とし込む関数が使用できます。

border_layout_panel_h関数の定義 eval <swing_obj> *Obj border_layout_panel_h @public -> <border_layout_panel_h maybe> ;;

このような関数はSwingオブジェクトごとに用意されており，型が一致すればそのハンドル返します。

*1：SgSwingが提供するSwingオブジェクトの多くは，SwingのクラスをSgSwingが継承して拡張したクラスのインスタンスです。
*2：「a_*_h」の形をした型は，SgSwingで抽象クラスとして扱われるクラスのインスタンスに対応するハンドルです。ただし，後述するように，Swingでは具象クラスのこともあります。
*3：派生クラスを持つクラスは，Javaでは抽象クラスにも具象クラスにすることができますが，SgSwingでは抽象クラスとして扱われます。
*4：Swingのパネル（javax.swing.JPanel）は具象クラスで，レイアウトマネージャを設定して使用しますが，SgSwingではそこまでの自由度を持たせず，レイアウトマネージャを固定した形でパネルを提供します。そのため，レイアウトマネージャのタイプごとにパネルの型が異なります。

3.5. リスナ

GUIに対するユーザの操作や，GUIの状態変化などの際にコールバックされる仕組みがリスナ（listener）です。 リスナは，コールバック単位で新しいアクタとして起動されます。

Swingでは，リスナの種類に応じてinterfaceが定義されており，プログラマはこれを実装して登録します。 発生するイベントに応じて呼び出されるメソッドが異なります。 SgSwingでは，呼び出される関数（リスナ実装関数：<event_proc>）の形は一種類のみで，渡されたイベント名により処理を切り替えるようプログラミングします。

リスナ実装関数 alias type <event_proc> @public := <context_h cstr event_info void fun> ;; alias type <event_info> @public := <cstr event_info_d "sango.util.map".map> ;;

第一引数には，コンテキストが渡されます。 このリスナ処理中は，渡されたコンテキストを使用します。

第二引数には，イベント名が渡されます。 例えば，SwingでのActionListener#actionPerformedの場合は"action.performed"になります（対応は「B. イベント」を参照）。

第三引数には，イベント情報が渡されます。 全てのイベントに対して，"source"というキーで，イベントが発生したSwingオブジェクトが(<swing_obj> obj_eid$)の形で格納されています。 それ以外の情報は，イベントによって異なります（詳細は「B. イベント」を参照）。

リスナを登録するには，リスナ実装関数からリスナ実装（<listener_h>）を作成し，それを対象のオブジェクトに登録します。

リスナ実装の作成 eval <event_proc> *Event_proc create_listener @public -> <listener_h> ;;

ボタンに対してActionListenerを登録する例を以下に示します。 登録関数は抽象ボタンが提供しますので，抽象ボタンに対して登録します。

リスナの登録例 \ <sw.context_h> *ECx <cstr> *E <sw.event_info> *EI -> <void> { do_something } sw.create_listener = *Lis, Cx (Button sw.as_abutton) Lis sw.add_action_listener

リスナの処理を複数ののアクタで分担して行うことも可能です。 この場合，Swing機能の呼び出しは直列化され逐次実行されますので，デッドロックが発生しないよう注意が必要です。 また，コールバックで起動された関数が終了（復帰）するとコンテキストが無効になるため，関連するすべての処理の完了を待ち合わせてから終了するようにします。

［目次］

3.6. プログラム終了

プログラムの終了にあたり，SgSwingを終了させる特別な処理は必要ありません。

リスナの処理としてプログラムを終了させる際には，"sango.system.runtime".exitではなく，"sango.system.runtime".shutdownを使用します。 exitは内部で例外を発生させアクタを強制終了させるため，コールバックが正常終了しなくなります。

フレームの閉じるボタンでプログラムを終了させるには，フレームのset_default_close_operationでexit_on_frame_close$を設定しておく方法もあります。
